# Тестирование конфигурации «Учёт в детейлинг-центре»

## Подход к тестированию

В экосистеме 1С:Предприятие тестирование реализуется через несколько инструментов и подходов.

---

## Инструменты тестирования

### 1. xUnit для 1С (vanessa-automation)

Популярный фреймворк модульного тестирования для 1С.  
Репозиторий: https://github.com/xDrivenDevelopment/xUnitFor1C

Пример структуры тестового модуля:

```bsl
// Тест_ПолучитьЦенуУслуги_ВозвращаетНоль_ЕслиЦенаНеНайдена
Процедура Тест_ПолучитьЦенуУслуги_ВозвращаетНоль() Экспорт
	
	НесуществующаяУслуга = Справочники.Услуги.ПустаяСсылка();
	Цена = ОбщийМодульСервер.ПолучитьЦенуУслуги(НесуществующаяУслуга, ТекущаяДата());
	
	Юнит.ПроверитьРавенство(0, Цена, "Цена должна быть 0 для несуществующей услуги");
	
КонецПроцедуры
```

### 2. Vanessa-ADD (Acceptance Driven Development)

Инструмент для приёмочного тестирования в стиле BDD.  
Репозиторий: https://github.com/vanessa-opensource/add

Сценарии описываются на языке Gherkin (русский):

```gherkin
# language: ru
Функционал: Запись клиента на услугу
  Как менеджер детейлинг-центра
  Я хочу создавать записи клиентов на услуги
  Чтобы вести учёт заявок

  Сценарий: Успешная запись клиента на мойку кузова
    Дано Клиент "Иванов Иван" с телефоном "+79001234567"
    И услуга "Мойка кузова" с ценой 1500 рублей
    Когда создаётся документ "Запись на услугу"
    И в табличную часть добавляется строка с услугой "Мойка кузова" в количестве 1
    Тогда сумма строки должна составить 1500 рублей
    И документ должен успешно провестись
```

### 3. Ручное тестирование — чек-лист

#### Справочники
- [ ] Создание клиента с заполнением всех реквизитов
- [ ] Попытка сохранить клиента без телефона → ожидается ошибка
- [ ] Создание услуги с указанием категории и времени выполнения
- [ ] Создание материала с единицей измерения
- [ ] Создание сотрудника

#### Регистр сведений «Цены услуг»
- [ ] Ввод цены для услуги «Мойка кузова» = 1500 руб. на дату 01.01.2024
- [ ] Проверка срабатывания срез последних при открытии формы документа

#### Документ «Запись на услугу»
- [ ] Создание документа, выбор клиента и сотрудника
- [ ] Добавление строки в ТЧ «Услуги» → автоподстановка цены
- [ ] Изменение количества → пересчёт суммы
- [ ] Проведение документа → движения в регистре «Выручка»

#### Документ «Поступление материалов»
- [ ] Оформление прихода 10 л шампуня
- [ ] После проведения: остаток = 10 л

#### Документ «Оказание услуги»
- [ ] Создание на основании «Записи на услугу»
- [ ] Добавление расхода 2 л шампуня
- [ ] Проведение → остаток = 8 л, движение в «Выручке»
- [ ] Попытка списать 100 л при остатке 8 л → ожидается сообщение об ошибке

#### Отчёты
- [ ] Отчёт «Выручка за период» за текущий месяц → показывает данные
- [ ] Отчёт «Остатки материалов» → актуальные остатки

## Тестовые данные

Скрипт начального заполнения (выполнить в консоли запросов или через обработку):

```bsl
// Создание тестового клиента
НовыйКлиент = Справочники.Клиенты.СоздатьЭлемент();
НовыйКлиент.Наименование    = "Иванов Иван Иванович";
НовыйКлиент.Телефон         = "+79001234567";
НовыйКлиент.Автомобиль      = "Toyota Camry";
НовыйКлиент.ГосНомер        = "А123БВ77";
НовыйКлиент.ДатаРегистрации = ТекущаяДата();
НовыйКлиент.Записать();

// Создание тестового сотрудника
НовыйСотрудник = Справочники.Сотрудники.СоздатьЭлемент();
НовыйСотрудник.Наименование = "Петров Пётр";
НовыйСотрудник.Должность    = "Мастер-детейлер";
НовыйСотрудник.Телефон      = "+79007654321";
НовыйСотрудник.Записать();

// Установка цены на услугу «Мойка кузова»
НоваяЗапись = РегистрыСведений.ЦеныУслуг.СоздатьМенеджерЗаписи();
НоваяЗапись.Период = НачалоДня(ТекущаяДата());
НоваяЗапись.Услуга = Справочники.Услуги.МойкаКузова;
НоваяЗапись.Цена   = 1500;
НоваяЗапись.Записать();
```

## CI/CD

Для автоматического запуска тестов рекомендуется использовать:
- **1С:Исполнитель** (OneScript) — https://oscript.io
- **Vanessa Runner** — запуск сценариев Vanessa-ADD из командной строки
- **GitHub Actions** с использованием Docker-образа 1С

Пример команды запуска тестов через 1С:Исполнитель:
```bash
oscript vanessa-runner.os --settings ./tests/vanessa-settings.json
```
